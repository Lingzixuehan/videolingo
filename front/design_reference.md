## 一、整体信息架构：有哪些大界面？

结合需求（讨论区、平台爬取、性能监控、视频分享等已暂时搁置/弱化），客户端可以分成六大模块：

1. **首页 / 概览（Dashboard）**
2. **视频与字幕模块**
   - 本地视频列表页
   - 单个视频学习页（视频播放 + 字幕 + 标注）
   - 字幕管理/编辑弹窗页
3. **学习模块**
   - 学习卡片复习页
   - 学习卡片管理页
4. **用户与隐私模块**
   - 登录 / 注册 / 忘记密码页
   - 个人资料 / 账号设置页
   - 数据与隐私页（导出、清除本地数据）
   - 学习计划页（每日目标与提醒）
5. **词库 / 词汇解释模块**
   - 在客户端主要是「词汇详情弹层/侧边栏」，独立界面很轻
6. **（可选）管理员/版主后台独立 Web，不在桌面端实现太多**
   - 按你们和甲方的约定：管理员可以删除用户，但不能修改；看不到邮箱与密码
   - 这个更适合做成独立 Web 后台，不需要集成在 Electron 主应用中

桌面端（Electron）最核心的是前四块：**首页、视频学习、学习卡片、个人与隐私**。

## 二、核心导航设计：用户怎么在界面间跳转？

建议主导航采用「左侧垂直菜单 + 顶部标题区域」的典型桌面管理系统布局：

左侧主菜单（固定）：

- 概览（首页）
- 我的视频
- 学习
  - 学习复习
  - 卡片管理
- 账号与设置
  - 个人资料
  - 学习计划
  - 数据与隐私
- （预留）后台管理入口（仅管理员显示）

这样：
- 用户日常主要走的路径：  
  「我的视频 → 选择一个视频 → 视频学习页 → 收藏/生成卡片 → 学习 → 复习」
- 设置类东西集中到「账号与设置」下面，不打扰主流程。

## 三、逐个界面说明：每个界面需要有哪些东西？

### 1. 首页 / 概览页（Dashboard）

**目标：** 给用户一个「我在这个应用里都干了啥/能干啥」的总览，让新用户知道入口，老用户看到进度。

**核心元素：**

1. 顶部欢迎区
   - 昵称 / 欢迎语：「早上好，XXX」
   - 今天日期、当前学习计划概要（例如：今日目标 20 张卡片，已完成 5 张）

2. 关键统计卡片（Stat Cards）
   - 已导入视频数
   - 已生成的学习卡片总数 / 今日待复习卡片数
   - 累计学习时长 / 连续学习天数
   -（如果要满足 SRS 的学习统计可视化）可放一小个「7 日学习趋势图」

3. 快捷操作区（大按钮）
   - 「上传视频」
   - 「开始学习（跳到今日复习队列）」
   - 「我的视频列表」
   - 「学习计划设置」入口

4. 最近活动 / 推荐
   - 最近观看的视频列表（缩略图 + 标题 + 上次观看时间 + 继续学习按钮）
   - 最近生成的卡片（几个缩略卡片，提供「立即复习」入口）

**与其他界面联动：**
- 「上传视频」→ 跳转到「我的视频」并打开文件选择对话框。
- 「开始学习」→ 跳到「学习复习」页，并加载今日卡片。
- 「最近视频」点击 → 直接进入对应的「视频学习页」。

---

### 2. 我的视频页（本地视频列表）

**目标：** 管理本地视频，体现「本地处理 + 服务器学习数据」的架构。

**核心元素：**

1. 顶部工具栏
   - 「添加视频」按钮（选择本地文件）
   - 搜索框（按标题模糊搜索）
   - 筛选：按字幕状态（未生成字幕 / 已有字幕）、按上传时间排序

2. 视频列表（列表或卡片视图）
   - 每条视频显示：
     - 缩略图 + 标题
     - 时长、大小、上传时间
     - 处理状态：未提取字幕 / 提取中 / 已有字幕
     - 标注数量（有几条笔记）
   - 操作按钮：
     - 「进入学习」→ 视频学习页
     - 「生成/重新生成字幕」
     - 「管理字幕」→ 字幕管理弹窗
     - 「删除视频」（确认弹窗，需要明确提示会删除本地字幕与笔记）

3. 上传流程
   - 选择文件后弹合规确认框：
     - 文本说明 + 「我确认拥有学习使用权」勾选
   - 进度条展示导入进度（可以假上传：因是本地文件，实际上主要是校验和入库）
   - 上传完成后出现在列表中

**联动：**
- 点击「进入学习」进入「视频学习页」并自动加载字幕、标注等。
- 「生成字幕」调用后台 API，状态更新后刷新列表。

---

### 3. 视频学习页（核心界面）

这个是整个产品体验的核心，需要重点设计。

**整体布局推荐：**

- 左：视频播放器（占据主要区域）
- 右：学习侧边栏（多 Tab：字幕 / 单词 / 短语 / 语法 / 笔记 / 卡片）
- 下：底端同步字幕（可选中/英/双语）

**需要的功能块：**

1. 视频播放器区域
   - 基本控制：播放/暂停、进度条、音量、倍速、全屏
   - 当前时间 / 总时长
   - 「添加标记」按钮：点击时暂停，并弹出笔记输入框

2. 底端同步字幕条
   - 显示当前句子字幕（可双语）
   - 每个单词可点击
   - 采用高对比度但不刺眼的背景（略透明黑底 + 白字）

3. 侧边栏 Tab 设计（建议右侧）

   - Tab1：**滚动字幕**
     - 按句列表显示
     - 当前句高亮，随播放滚动
     - 点击某句 → 视频跳转到该句时间点
     - 右侧每句可有小图标：标记、语法分析入口

   - Tab2：**单词 / 词汇**
     - 当前视频出现过的单词列表（按出现顺序或按字母排序）
     - 每个词条：
       - 单词、音标、词性、中文释义
       - 来源标签：例如「四级词」「六级词」「托福词」——按你们约定，展示词库来源，不做「词汇量过滤」
       - 「收藏」按钮：点击生成/关联学习卡片
     - 点击某词 → 展开详细解释，例句、原句上下文

   - Tab3：**短语 / 俚语**
     - 类似单词列表，但按短语/俚语
     - 提供收藏功能

   - Tab4：**长难句语法分析**
     - 当前句子的结构图（树状结构或分行标注）
     - 提示主谓宾 / 从句关系 / 关键语法点
     - 支持用户点「生成卡片」：把该句 + 解析内容生成一张语法卡片

   - Tab5：**笔记 / 标注**
     - 显示用户在此视频的所有标注（按时间排序）
     - 每条：
       - 时间戳、简要内容
       - 点击 → 视频跳转
       - 编辑、删除按钮（删除可撤销一段时间）
       - 「生成卡片」按钮

   - Tab6（可选）：**本视频相关卡片**
     - 列出所有来自该视频的学习卡片，可以直接进入复习或编辑

4. 单词解释弹出层
   - 用户点击底部字幕中的单词时，在字幕附近或播放器上方弹出一个小浮层：
     - 单词 + 音标 + 词性 + 中文释义
     - 来源标签（四级/六级等）
     - 「收藏为卡片」按钮

**联动：**
- 底部字幕 → 侧边栏滚动字幕：互相同步。
- 点击单词 → 调用词库 → 可以直接生成卡片 → 「学习」模块。
- 添加标注 → 自动生成学习卡片（后台），但前端可以给立即反馈。

---

### 4. 字幕管理页 / 弹窗

从「我的视频」或「视频学习页」进入。

**内容：**

1. 字幕版本列表
   - 列出当前视频的所有字幕版本：
     - 来源：自动生成 / 用户上传
     - 语言：英文 / 中文 / 双语
     - 创建时间
   - 默认字幕的标记

2. 字幕操作
   - 「设为默认」
   - 「下载字幕文件」
   - 「编辑字幕」→ 打开字幕编辑器

3. 字幕编辑器（单独大弹窗）
   - 左侧：字幕句子列表（序号 + 起止时间 + 文本）
   - 中间：可编辑区域（支持逐句编辑）
   - 右侧：简单操作：
     - 合并选中两句
     - 拆分当前一句
   - 上方：保存 / 另存为新版本 / 取消

**联动：**
- 保存后回视频学习页时使用最新版本。
- 下载时生成 SRT/VTT / JSON 等格式。

### 5. 学习复习页（间隔重复卡片）

**目标：** 实现 SRS 中的「卡片回顾」用例，操作简单、节奏感强。

**布局：**

- 中心大卡片：类似 Anki / Quizlet
- 底部三个按钮：困难 / 一般 / 简单
- 顶部：进度与统计

**内容：**

1. 顶部信息栏
   - 今日目标：目标张数，已完成，剩余
   - 当前卡片索引：例如「5/20」
   - 可选：显示来源视频的小缩略图 + 时间戳

2. 卡片呈现
   - 正面默认展示「问题」：例如英文句子/单词
   - 「翻转」按钮：显示答案/中文释义/笔记
   - 支持键盘快捷键（Space 翻面，1/2/3 对应三种难度）

3. 反馈按钮
   - 困难 / 一般 / 简单 3 个凸显的按钮
   - 点击后直接跳到下一张卡片

4. 右侧/顶部轻量工具
   - 「编辑卡片」按钮（跳转到卡片管理页并定位）
   - 「查看来源」→ 调用视频学习页，并定位到该卡片来源的时间点

---

### 6. 学习卡片管理页

**目标：** 对卡片做增删改查/导入导出。

**内容：**

1. 筛选区
   - 搜索框（按单词/句子内容）
   - 按来源视频筛选
   - 按掌握度 / 标签筛选

2. 卡片列表（可列表或卡片网格）
   - 显示：
     - 正面摘要
     - 类型（单词卡 / 短语卡 / 句子卡 / 语法卡 / 笔记卡）
     - 掌握度（难 / 一般 / 易）
     - 上次复习时间 / 下次复习时间
   - 操作：
     - 查看/编辑
     - 删除
     - 「加入今日复习」

3. 导入导出
   - 「导出选中卡片」→ JSON/CSV
   - 「导入卡片」→ 选择文件，导入到当前用户库中

### 7. 登录 / 注册 / 忘记密码

你们有 Electron + 后端 API + 邮箱验证。

**改动点（和甲方约定）：**
- 优先邮箱绑定，不急着做第三方登录和短信。
- 管理员可以删除但不能修改用户，客户端一般只给普通用户用，管理员用后台 Web。

**登录页内容：**

- 左侧可放品牌插画 / 简短产品介绍
- 右侧登录表单：
  - 邮箱
  - 密码
  - 「登录」按钮
  - 「注册新账号」、「忘记密码」链接

**注册页：**

- 邮箱
- 密码 / 确认密码
- 昵称
- 勾选隐私协议
- 「发送验证码」按钮 → 收到验证码 → 输入 → 注册完成

**忘记密码：**

- 邮箱输入
- 「发送重置链接」或「发送验证码」
- 输入新密码

界面上不需要过度复杂，但要有清晰错误提示与状态反馈。

### 8. 个人资料 / 账号设置页

**内容：**

1. 个人资料信息

2. 学习偏好设置

3. 安全设置
   - 修改密码入口（跳转到修改密码表单）
   - 显示当前登录设备（可选）

### 9. 数据与隐私页

对应 SRS 的「数据导出/备份」「数据保留与隐私设置」。

**内容：**

   - 警告说明（红色）

## 四、界面之间的典型用户路径

把复杂需求串成几个典型使用场景：

### 场景 1：新用户从零开始

1. 打开应用 → 登录/注册
2. 跳转到首页 → 没有视频 → 显示「去上传第一个视频」
3. 用户点击「上传视频」 → 到「我的视频」 → 导入本地视频 → 合规确认
4. 在视频列表中点击「生成字幕」
5. 字幕生成完成后 → 点击「进入学习」
6. 在视频学习页播放视频、查看底部字幕，点击某个难词 → 弹出解释 → 收藏生成卡片
7. 切换右侧「笔记/标注」Tab → 添加一个笔记 → 自动生成卡片
8. 回首页 → 点击「开始学习」 → 打开学习复习页，看到刚才生成的卡片

### 场景 2：老用户日常复习

1. 启动应用 → 首页看到「今天待复习 23 张」
2. 点击「开始学习」 → 学习复习页
3. 完成复习 → 系统提醒「今日目标已完成」，首页数据更新

### 场景 3：清理数据与隐私管理

1. 用户担心占用空间 → 去「数据与隐私」
2. 将数据保留策略改成 30 天
3. 导出学习卡片备份
4. 点击「清除本地数据」→ 输入密码 → 删除本地视频与缓存字幕

## 五、整体美术与交互风格建议

甲方要求：界面简洁美观、风格统一、结构清晰，杜绝抽象猎奇。

**1. 视觉风格：轻量教育工具风**

- 色彩：
  - 主色：冷静、偏蓝/青（如 #2563EB 或 #22A6B3 这种）
  - 辅助高亮：稍暖（橙色/黄色）用在按钮和进度上
  - 背景：浅灰/近白，避免纯白刺眼（#F5F5F5）

- 字体与排版：
  - 统一使用系统无衬线字体（中文：默认系统 + 微软雅黑/思源黑体）
  - 标题层次分明：H1/H2/H3 的字号与粗细要有明显区别
  - 文案尽量短、清晰，偏工具说明而不是营销话术

- 图标与插图：
  - 简单线性图标（播放、学习、统计、设置等）
  - 插画少用或轻量，用在登录/注册页和首页空状态即可

**2. 交互风格：偏「桌面应用」而不是「网页」**

- 多使用「对话框/弹窗」而不是新页面：比如字幕编辑、删除确认。
- 善用右键菜单：视频列表中的右键可以提供「删除、重命名、打开所在路径」等。
- 键盘快捷键：
  - 空格：播放/暂停
  - 上下左右：快进/快退、切卡片等

**3. 统一组件风格**

- 所有按钮统一样式：主按钮（实心，主色）、次按钮（线框）、危险按钮（红色）
- 表单元素统一 padding、圆角、错误提示方式
- 卡片 UI：用于 StatCard、学习卡片预览等，统一阴影/圆角

**4. 国际化准备**

- 所有文本都从 i18n 文案文件中取，不硬编码到组件。
- 中英文切换放在「个人资料 / 设置」中，但首期只需要保证文案分离即可。
